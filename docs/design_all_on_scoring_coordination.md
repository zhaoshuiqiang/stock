# 设计文档：全开场景评分修正协调（强势趋势护栏——验证后未采纳）

> 版本：v4.19.20260723 ｜ 状态：已验证，结论=不采纳 ｜ 作者：AI 协作

## 1. 背景

13 个实验开关全开时为“未联合校准的修正叠加态”：5 个抑制趋势/追高开关
（deemphasizeTrendStrength、deemphasizeBreakoutChase、useShortTermTrendDiscount、
useShortTermRealtimeReprofile、useRecalibratedDirection）在信号/技术/共振/方向/实时
多层重复扣分。**初始假设**：这会系统性压制“健康强势上升趋势”个股，把它们拖出看多带，
与打板/主线选股目标相悖，故拟加“强势趋势护栏”把这类个股方向分下限抬回 +12。

## 2. 验证（scripts/analyze_strong_trend_guard.py）

数据：10 份留档 CSV 去重 3281 行；口径：topSignals 含均线多头/趋势强度，
留档涨跌幅界定“温和上涨/抛物线”，价格变动(%) 为前向收益（浮动，非固定周期，近似）。

| 子集 | n | 前向均值 | 上涨胜率 |
|---|---|---|---|
| 全样本基线 | 3281 | -0.29% | 50.4% |
| 均线多头/趋势强度(广义) | 1211 | -1.02% | 43.3% |
| 护栏目标(叠加+温和上涨0~7%) | 629 | -1.25% | 41.7% |
| 其中“非买入”子集(护栏真正救援对象) | 345 | -1.13% | 43.8% |
| 被排除的抛物线(≥7%) | 51 | +0.88% | 52.9% |

## 3. 结论：不采纳

护栏目标population**前向收益为负**（均值 -1.1%~-1.25%、胜率约 42%），与既有实证
“均线多头前向 -2.0%、趋势强度 -0.70%”一致。即：全开下“看似被过度压制”的强势股，
**实际并未跑赢**；把它们上抬回看多带（+12，谨慎买入级）会**重新拉高亏损样本**，
与护栏初衷相反。对一个“默认即生效”的护栏，正向 edge 的举证不成立，且有负向证据。

据此撤回护栏，评分/方向行为与验证前**字节等价**（已用 git diff HEAD~1 核验）。

局限：CSV 缺 ADX/bias/rsi，切片是护栏真实人群的**超集近似**；且为浮动收益。
但所有可测切片一致为负，方向明确。

## 4. 后续正确路径（如仍要做救援）

1. 用 decision_outcomes 的**固定 1/3/5 日**命中率（而非浮动 CSV）复核；
2. 必须先**数据定位**真正具备正向 edge 的子人群（可能需 regime 门控：牛市趋势 vs 结构分化），
   而非人工加条件；
3. 任何救援都应**默认关 + 灰度**，遵循本仓“validated 前默认关”的既有约定；
4. 全开场景的更优解可能是“修正封顶/命名预设”，但同样需先证明存在被过度压制的正 edge 人群。
